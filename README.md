# TaskFlow — Full Stack Task Manager

TaskFlow is a simple full-stack task manager built using React, Node.js, and MongoDB.
The project focuses on implementing a complete application with a frontend, backend, and database.

## Features

* Add tasks with title, description, priority, due date, and time
* Mark tasks as complete or incomplete
* Delete tasks
* Filter tasks (All / Active / Completed)
* Responsive layout for both desktop and mobile
* Data is stored in MongoDB and persists after refresh

## Technologies Used

Frontend: React (with Vite), Axios
Backend: Node.js, Express
Database: MongoDB Atlas with Mongoose

## Database Schema

Collection: `tasks`

Fields used:

* `title` (String, required)
* `description` (String, optional)
* `priority` (String: low / medium / high)
* `completed` (Boolean, default false)
* `dueDate` (Date, optional)
* `dueTime` (String, optional)
* `createdAt`, `updatedAt` (auto-generated by Mongoose)

## API Endpoints

Base URL (Local): http://localhost:5000/api  
Base URL (Production): https://taskflow-backend-sg4b.onrender.com/api

* GET `/tasks` → fetch all tasks
* POST `/tasks` → create a new task
* PATCH `/tasks/:id/toggle` → toggle completion
* DELETE `/tasks/:id` → delete a task

Example POST request:

{
"title": "Study for exam",
"description": "Cover chapters 4 and 5",
"priority": "high",
"dueDate": "2025-04-10",
"dueTime": "14:00"
}

## Setup Instructions

1. Clone the repository
   git clone https://github.com/sheema-salah/taskflow.git

2. Backend setup
   cd backend
   npm install

    Create a `.env` file and add:
    PORT=5000
    MONGO_URI=your_connection_string

    Run the server:
    npm run dev

3. Frontend setup
   cd ../frontend
   npm install
   npm run dev

4. Open in browser
   http://localhost:5173

## Deployment

The frontend is deployed on Vercel and the backend on Render.

## Approach

The project was developed by first designing the database schema and implementing backend API endpoints.
After verifying the API functionality, the frontend was built and integrated with the backend.
The focus was on maintaining a simple structure and ensuring usability across different screen sizes.

Create a `.env` file inside the `frontend` folder and add:

If running locally without this, the app will default to:
http://localhost:5000/api
